# Topic Generation

Quickly generate 3–4 WeChat article topic directions, each including a title, outline, and pros/cons analysis.

## When to Use

- After saving a brief
- When the user says "give me some topic ideas" or "suggest a few directions"
- Step 3 of the article creation workflow (topic discussion)

## Core Features

1. **Provide 3–4 topic directions**: Each from a different angle
2. **Each topic includes**: Title, core angle, outline, workload estimate, pros/cons analysis
3. **Wait for user selection**: Do not assume which one they will choose

## Topic Direction Types

### 1. In-Depth Review

- **Angle**: Comprehensive testing + data comparison
- **Suitable for**: New products, new technologies
- **Workload**: ⭐⭐⭐ (requires real testing)

### 2. Practical Tutorial

- **Angle**: Step-by-step guide + replicable
- **Suitable for**: Tool usage, technical practice
- **Workload**: ⭐⭐ (requires step verification)

### 3. Insight & Opinion

- **Angle**: Unique perspective + deep thinking
- **Suitable for**: Industry trends, methodology
- **Workload**: ⭐ (primarily driven by thinking)

### 4. Case Study Breakdown

- **Angle**: Success case + method extraction
- **Suitable for**: Growth cases, product cases
- **Workload**: ⭐⭐ (requires case gathering)

## Topic Template

```markdown
## Topic 1: [Compelling Title]
**Core Angle**: xxx
**Workload Estimate**: ⭐⭐ (Medium)
**Does it require real testing**: Yes / No

### Outline
1. Opening: xxx
2. Core Section 1: xxx
3. Core Section 2: xxx
4. Closing: xxx

**Estimated Word Count**: 3,000 words

### Advantages
- ✅ Advantage 1
- ✅ Advantage 2

### Disadvantages
- ⚠️ Disadvantage 1
- ⚠️ Disadvantage 2
```

## Title Strategies

**Compelling title formulas**:

- Comparison type: "A vs B — How Big Is the Gap?"
- Pain point type: "Why Does XX Always Fail?"
- Results type: "I Used XX and Achieved YY in 3 Months"
- Reveal type: "The Truth Behind XX"
- List type: "5 Methods That Will XXX"

## Notes

- Each topic angle must be clearly different
- Pros/cons analysis should be honest (don't exaggerate)
- Workload estimates should be accurate
- Wait for the user to choose — never assume
